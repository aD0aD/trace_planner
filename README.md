# Trace Planner（轨迹拼接与分析工具）

一个“打开就能用”的轨迹小工具：  
导入 `KML/KMZ/GPX`，拼接路线、看分段数据、加标注、看海拔图、再一键导出。

适合徒步复盘、反穿规划、活动线路整理、做路线海报。

---

## 这工具能干嘛？

- **多轨迹导入**：`KML / KMZ / GPX` 都支持
- **地图切换**：卫星图、地形图、街道图 + 阴影地形
- **轨迹拼接**：从不同轨迹截取片段，自由排序、反向拼接
- **分段分析**：
  - 起终点滑块截段（距离/爬升/下降）
  - 断点分段（手动断点 + 自动按 N 段均分）
- **实时海拔图**：
  - 起点/终点位置同步移动
  - 点旁显示当前海拔
  - 纵轴显示海拔刻度
- **标注系统**：
  - 多类型多颜色（起点/终点/营地/补给/风险/普通）
  - 标签可拖拽调整，避免重叠
- **导出能力**：
  - 轨迹导出：`KML / GPX / GeoJSON`
  - 海拔剖面图：在「轨迹分析工具」里一键导出 `PNG`（与页面实时高度图同源数据）
  - 工程保存：`project.json`
- **爬升算法可选**：
  - 原始累计
  - 平滑累计（无阈值）
  - 平滑 + 阈值累计（默认）

---

## 仓库里有什么？

就 **一个程序文件**：`Trace_planner.html`（另有 `LICENSE`）。  
海拔剖面、实时高度图、导出爬升下降图 PNG 都在浏览器里完成，**不需要 Python**。

如果你本地有很多 `KML/KMZ` 素材，建议把本仓库单独放在一个文件夹里（例如 `trace_planner/`），和轨迹、PPT、截图分开，互不干扰。

---

## 3 步快速上手

1. 下载或克隆仓库  
2. 双击打开 `Trace_planner.html`  
3. 导入轨迹，开始拼接和分析

> 纯前端，零依赖；想本地起一个静态服务也可以，不是必须。

可选（本地 HTTP）：

```bash
cd <repo>
python3 -m http.server 8000
```

然后访问：`http://localhost:8000/Trace_planner.html`

---

## 推荐使用流程

1. 导入多条轨迹（`KML/KMZ/GPX`）
2. 选择“当前分析轨迹”
3. 用滑块截段，或者加入拼接篮子做新路线
4. 设置断点分段，查看每段里程与爬升下降
5. 补充标注（营地、补给点、风险点等）
6. 导出轨迹/海拔图，或保存工程下次继续

---

## 输入与输出格式

- **输入**
  - `KML`（支持 `LineString` / `gx:Track`）
  - `KMZ`（自动解压读取内部 KML）
  - `GPX`（`trkpt`）

- **输出**
  - `KML`（轨迹 + 标注）
  - `GPX`（轨迹 + waypoint 标注）
  - `GeoJSON`（LineString + Point）
  - `PNG`（海拔剖面图）
  - `project.json`（完整工程状态）

## 海拔算法怎么选？

不同设备记录海拔的“抖动”差异很大，所以给了多种模式：

- **原始累计**：最灵敏，也最容易把噪声算成爬升
- **平滑累计**：比较均衡，常用推荐
- **平滑+阈值**：更抗噪，适合点很密或海拔抖动明显的轨迹

可以在界面中改平滑窗口和阈值，统计会实时更新。

## 小提醒

- 地图底图依赖在线瓦片，需要网络
- 轨迹点非常多时，老设备可能会卡一点
- 同一路线来自不同设备，爬升结果有差异是正常现象
